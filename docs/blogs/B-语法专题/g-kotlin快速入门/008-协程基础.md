### 第02节 定义协程

在 ViewModel 当中定义协程

```kotlin
import android.util.Log
import androidx.lifecycle.MutableLiveData
import androidx.lifecycle.ViewModel
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.SupervisorJob
import kotlinx.coroutines.delay
import kotlinx.coroutines.launch
import kotlinx.coroutines.withContext

class DemoViewModel : ViewModel() {

    private val TAG = DemoViewModel::class.simpleName

    // 创建了 LiveData 的对象
    private var liveDataVersionBean = MutableLiveData<List<VersionBean>>()


    // CoroutineScope 是协程的访问范围
    // by lazy 是 Kotlin 提供的一种属性委托（Property Delegation）机制。
    // 通过使用 by lazy，我们可以将一个属性的初始化逻辑延迟到第一次被访问时才执行。这在某些场景下非常有用，比如：
    // 1、初始化操作比较耗时（如网络请求、数据库查询等）
    // 2、属性依赖于其他尚未初始化的对象
    // 3、想避免不必要的初始化开销
    protected val viewModelScope: CoroutineScope by lazy {
        // 创建一个与 ViewModel 生命周期绑定的 Scope
        CoroutineScope(SupervisorJob() + Dispatchers.Main)
    }


    // 访问数据的方式
    fun fetchData() {
        // 启动协程在 IO 线程上
        viewModelScope.launch(Dispatchers.IO) {
            // 延迟 2秒钟执行
            delay(2000L)
            // 模拟网络访问的结果 获取到网络接口上面的数据。 webInterfaceData 表示模拟的网络接口数据访问
            val result: List<VersionBean> = webInterfaceData()
            // 可选方案一: 将数据装载到 LiveData 上面
            liveDataVersionBean.postValue(result)
            // 可选方案二: 直接切换到 UI 主线程上, 进行数据输出或者UI更新操作
            withContext(Dispatchers.Main) {
                Log.i(TAG, "result: $result")
            }
        }
    }
}
```

只需要在 Activity 或者 Fragment 当中调用即可

```kotlin
viewModle.fetchData()

viewModle.liveDataVersionBean....
```











### 第03节 关于ViewModelScope

`viewModelScope` 是什么？

```
来源说明:
	1、ViewModelScope 是一个  CoroutineScope 的实例，它是通过 ViewModel 和 ViewModelScope 属性获取到的。
	2、它是 Android Jetpack 当中 ViewModel 类提供的一个协程的作用域。（coroutine scope） 
	3、它能够与 ViewModel 的生命周期绑定, 确保在 ViewModel 被清除的时候, 自动取消所有 正在运行的协程。
	
核心作用：
	1、生命周期感知。  当ViewModel 被销毁的时候（Activity或者Fragment被释放时）ViewModel自动取消所有的协程。
	2、默认调度器。    如果不指定调度器，默认使用的是 Dispatchers.Main 主线程，但是我们可以通过 launch 方法显示指定其他的调度器
	3、线程安全。		我们可以i在任意线程当中启动协程，通常会在 IO线程执行耗时操作，然后切换回主线程更新 UI
```

为什么需要使用 `viewModelScope`

```
1、避免内存泄漏
2、生命周期一致性
3、简化代码
```

对比 `lifecycleScope` 虽然 `viewModelScope` 和 `lifecycleScope` 都是生命周期感知的协程作用域，但是他们的使用场景略有不同。

| 特性             | `viewModelScope`       | `lifecycleScope`                                |
| ---------------- | ---------------------- | ----------------------------------------------- |
| 所属对象         | `ViewModel`            | `Activity`  或者 `Fragment`                     |
| 生命周期绑定     | `ViewModel` 的生命周期 | `Activity` 或者 `Fragment` 的生命周期           |
| 是否适合长期任务 | 适合                   | 不建议 （可能在 Activity 销毁后，还在继续运行） |













### 第04节 总结说明

常见的协程调度器

| 调度器                   | 用途                                   |
| ------------------------ | -------------------------------------- |
| `Dispatchers.Main`       | 主线程（UI线程）                       |
| `Dispatchers.IO`         | IO     密集型任务（如 网络、磁盘读写） |
| `Dispatchers.Default`    | CPU 密集型任务（如 计算）              |
| `Dispatchers.Unconfined` | 不限制线程 （用于调试）                |

Kotlin 协程为 Android 开发提供了一种优雅的异步编程方式，相比传统的  `AsyncTask` 或  `Handler` 协程具有更强的可读性 和 更简洁的语法。

通过合理使用 `viewModelScope`、`lifecycleScope` 和 不同的调度器，开发者可以构建出高效、稳定而且易于维护的异步逻辑。















## 第二章 结合Flow

### 第01节 基础介绍

协程与 Flow 的结合使用，是现代 Android 开发当中，处理 `异步数据流` 的一种优雅的方式，他们共同提供了: 

```
1、非阻塞的异步操作
2、更清晰的代码结构
3、更好的错误处理机制
4、与 Android 生命周期良好集成
```

其中 Flow 还包含有一些高级的用法

```
1、flatMapMerge
2、ofEach
3、conflate 
```

基础功能点介绍

| 功能                    | 说明                                          |
| ----------------------- | --------------------------------------------- |
| `flow { ... }`          | 创建一个 `Flow`  用于发射多个值               |
| `Collect { ....  }`     | 收集 `Flow` 中的值，通常在协程当中调用        |
| `viewModelScope.launch` | 在 `viewModel` 中启动协程，确保生命周期的安全 |
| `collectLatest`         | 收集 `Flow` 的最新值，避免重复更新 UI         |
| `StateFlow`             | 可观察的 `Flow`  适合与 UI 绑定               |

**1、数据模型**

定义 User 模型

```kotlin
data class User(val id: Int, val name: String)
```

**2、数据仓库**

定义 仓库数据访问 UserRepository

```kotlin
import kotlinx.coroutines.delay
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.flow

class UserRepository {
    
    // 模拟从网络上 获取用户的信息函数
    fun getUser(userId: Int): Flow<List<User>> = flow {
        // 模拟网络延迟
        delay(100)
        // 构建数据
        val list = ArrayList<User>().apply {
            add(User(userId, "Jone Doe"))
            add(User(userId, "Lily"))
            add(User(userId, "BankJ"))
        }

        if (userId < 0) {
            list.clear()
        }
        // 发射数据
        emit(list)
    }
}
```



**3、双向绑定**

在 ViewModel 当中的写法

```kotlin
import androidx.lifecycle.ViewModel
import kotlinx.coroutines.CoroutineScope
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.SupervisorJob
import kotlinx.coroutines.flow.MutableStateFlow
import kotlinx.coroutines.flow.StateFlow
import kotlinx.coroutines.launch

class DemoViewModel : ViewModel() {

    // 创建 UserRepository 的对象
    private val mUserRepository = UserRepository()

    private val mMutableStateFlow = MutableStateFlow<List<User>?>(null)

    // 创建 StateFlow 流对象
    val mStateFlowUserList: StateFlow<List<User>?> = mMutableStateFlow


    // CoroutineScope 是协程的访问范围
    // by lazy 是 Kotlin 提供的一种属性委托（Property Delegation）机制。
    // 通过使用 by lazy，我们可以将一个属性的初始化逻辑延迟到第一次被访问时才执行。这在某些场景下非常有用，比如：
    // 1、初始化操作比较耗时（如网络请求、数据库查询等）
    // 2、属性依赖于其他尚未初始化的对象
    // 3、想避免不必要的初始化开销
    protected val viewModelScope: CoroutineScope by lazy {
        // 创建一个与 ViewModel 生命周期绑定的 Scope
        CoroutineScope(SupervisorJob() + Dispatchers.Main)
    }


    // 通过 userId 加载用户数据
    fun loadUser(userId: Int) {
        // 启动协程在 IO 线程上
        viewModelScope.launch(Dispatchers.IO) {
            // 在这里去调用仓库的方法, 将结果进行收集
            mUserRepository.getUser(userId).collect { data ->
                // 收集的结果, 交给 MutableStateFlow 进行存储。
                // 1. 对外, 调用时, 采用的是 loadUser()
                // 2. 对外, 监听时, 采用的是 mStateFlowUserList
                mMutableStateFlow.value = data
            }
        }
    }
}
```



**4、界面UI**

在 Activity 的 onCreate 当中

```kotlin
class DemoActivity : AppCompatActivity() {

    private val TAG = DemoActivity::class.simpleName
    private lateinit var viewModel: DemoViewModel
    private lateinit var binding: ActivityDemoBinding

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        binding = ActivityDemoBinding.inflate(layoutInflater)
        setContentView(binding.root)
        viewModel = ViewModelProvider(this).get(DemoViewModel::class.java)
        binding.viewModel = viewModel
        binding.lifecycleOwner = this
 
        // 监听到数据的处理
        lifecycleScope.launch {
            viewModel.mStateFlowUserList.collectLatest {
                it?.let { users ->
                    Log.i(TAG, "users: $users")
                }
            }
        }

        // 调用方法, 访问数据
        viewModel.loadUser(1)
    }
}
```
