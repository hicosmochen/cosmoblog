## 第一章 四种空值处理方案

### 第01节 if语句处理

案例代码

```kotlin
// 定义数据
var data = null

// 逻辑判断
if (data == null) {
    Log.i(TAG, "data is null")
} else {
    Log.i(TAG, "data is not null")
}
```





### 第02节 when 语句处理

案例代码

```kotlin
// 定义数据
val data: Int? = null

// 逻辑判断
when (data) {
    null -> {
        Log.i(TAG, "data is null")
    }

    else -> {
        Log.i(TAG, "data is not null")
    }
}
```





### 第03节 组合符号

案例代码

```
// 定义数据
val data: Int? = null

// 逻辑判断
data?.let {
    Log.i(TAG, "data is not null")
} ?:  Log.i(TAG, "data is null")
```

通过  `?.let`  符号，判断是否为空：

1、如果不为空的情况下，执行 `let` 后面的语句块。 

2、如果为空的情况下， 执行 `?:`  后面的语句

3、这里的 `?:` 以及后面的语句可以省略







### 第04节 非空断言

案例代码

```kotlin
// 定义数据
val data: Int? = null

// 逻辑判断 
Log.i(TAG, "data is  ${data!!}")
```

这里断言 data 的数据值不为 `null`  采用的符号是 `!!` 断言

但是这里很显然 data 实际的数据值是 `null` 因此这里会出现  空指针异常 `NullPointerException`

所以，我们在使用的时候，尽量少用 非空断言 `!!`











## 第二章  推荐用法

### 第01节 推荐方案

```
我们推荐使用组合符号的方案：
	1、  ?.let{  }
	2、  ?:
```

说明1：

```
?.let{  }

上面的写法，可以进行空值的判断，如果当前的值是 空值，则不会执行大括号里面的内容
```

说明2：

```
?:

上面的写法，表示出现空值之后的应对方案，通常情况下，这种写法是给出现空值时，赋值默认值的操作
```





### 第02节 案例代码

代码

```kotlin
viewModel?.let {
    viewModel.helloworld()
}
```

说明

```
在使用 ViewModel 的过程中，可能出现空值的情况。

1、如果没有出现空值，就执行 helloworld方法。
2、如果出现了空值，那么就跳过这段代码，继续执行下面的代码。
```
