## 第一章 预先安装环境

### 第01节 硬件环境

基础硬件环境

```
编译AOSP需要强大硬件支撑：

1、CPU（建议8核以上）
2、大内存（至少16GB，推荐32GB）
3、高速大容量SSD（至少500GB）

官方文档中针对特定 Ubuntu 版本的依赖列表（如Ubuntu 18.04/20.04）

下载地址: https://ubuntu.com/desktop
```







### 第02节 准备REPO

使用 `repo`  需要安装 `python3` 以及  `git`

相关命令

```shell
# 环境准备 安装 repo 和 python3
sudo apt-get update
sudo apt-get install curl git python3


# 配置 git 信息
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
```







### 第03节 配置环境变量

配置环境变量

```shell
# 配置环境变量的方式
nano ~/.bashrc

# 将下面的这句话添加到最后末尾
# 按 Ctrl + O，（字母欧）然后按 Enter 键保存文件。
# 按 Ctrl + X，（字母艾克斯） 退出 nano 编辑器。
export REPO_URL='https://mirrors.bfsu.edu.cn/git/git-repo'


# 刷新环境变量
source ~/.bashrc
```











## 第二章 防断线脚本

### 第01节 创建脚本

命令

```shell
nano  download.sh
```





### 第02节 脚本内容

```shell
#!/bin/bash
echo "=== 开始同步 AOSP 源码 ==="
repo sync -c -j4
while [ $? -ne 0 ]; do
    echo "=== 同步失败，3秒后自动重试... ==="
    sleep 3
    repo sync -c -j4
done
echo "=== AOSP 源码下载完成！ ==="
```





### 第03节 退出保存

退出并且保存

```
1、退出，按下快捷键  Ctrl + O   字母欧
2、保存，按下快捷键  Ctrl + X   字母艾克斯
```

注意需要修改脚本为执行文件，使用下面的命令

```shell
chmod  +x  download.sh
```









## 第三章 下载源码

### 第01节 克隆镜像

命令

```shell
# 使用北京外国语大学镜像（推荐，也是由清华 TUNA 团队维护，但人少）
git   clone  https://mirrors.bfsu.edu.cn/git/git-repo repo
```





### 第02节 安卓10的源码

克隆 Android 10 的源码

```shell
repo   init   -u   https://mirrors.bfsu.edu.cn/git/AOSP/platform/manifest  -b  android-10.0.0_r41  --no-clone-bundle
```



### 第03节 启动脚本

将脚本复制到需要下载的目录下面，当然也可以在需要下载的目录下面创建脚本。

使用下面的命令，启动脚本，等待下载完毕。

```shell
./download.sh
```





### 第04节 运行结果

运行的结果如下 （我在虚拟机当中下载的时长约  `2小时40分钟`）

```shell

    repo sync -c -j4

```









## 第四章 编译源码

### 第01节 启动编译

启动编译的命令如下，注意前面有个点

```
.  build/envsetup.sh
```





### 第02节 选择编译版本

执行命令

```
lunch  
```

可以选择下面的环境

```
lunch   aosp_x86_64-eng
```





### 第03节 启动编译命令

执行命令

```
m  -j4
```

编译完毕之后，显示的内容是

```
#### build completed successfully .... ####
```

最终编译的文件产生在  `out/` 镜像当中，首次编译花费的时间较久。



> 下面就是等待编译完成，这个过程，大概需要  1 - 4 个小时不等，这个过程中，会出现各种各样的错误。
