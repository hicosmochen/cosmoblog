### 第02节 准备 Python

使用 `repo`  需要安装 `python3`

相关命令

```shell
# 环境准备 安装 python3
sudo apt-get update
sudo apt-get install curl git python3
sudo apt install python-is-python3


# 环境准备 安装 python2
# 因为旧的版本 Android-10 需要使用的是 python2 进行编译
sudo apt install python2
```

需要将 python 指向 python2 的环境

```shell
# 设置配置选项
sudo update-alternatives --install /usr/bin/python python /usr/bin/python2 1
sudo update-alternatives --install /usr/bin/python python /usr/bin/python3 2


# 切换默认配置项
# 执行后会看到类似下面的菜单，输入 python2 对应的编号（通常是 1 或 0）并回车
sudo update-alternatives --config python

# 验证配置
# 如果出现的结果是 Python 2.7.x，说明配置成功
python --version
```

### 第03节 创建启动脚本

下载过程中，容易出现断联的情况，建议提前创建好脚本，出现断联能够及时重连。

创建脚本文件

```shell
nano  download.sh
```

添加脚本的内容

```shell
#!/bin/bash
echo "=== 开始同步 AOSP 源码 ==="
repo sync -c -j4
while [ $? -ne 0 ]; do
    echo "=== 同步失败，3秒后自动重试... ==="
    sleep 3
    repo sync -c -j4
done
echo "=== AOSP 源码下载完成！ ==="
```

保存并且退出

```
1、保存，按下快捷键  Ctrl + O   字母欧
2、按下 回车键 Enter 切换到外部菜单
3、退出，按下快捷键  Ctrl + X   字母艾克斯
```

注意需要修改脚本为执行文件，使用下面的命令

```shell
chmod  +x  download.sh
```

启动脚本，使用下面的命令，等待下载完毕。 （ 我花费的时间约 2小时40分钟 ）

```shell
./download.sh
```

## 第四章 启动模拟器

### 第01节 编写脚本

在用户目录下面，常见一个脚本文件，用于启动模拟器

```shell
nano  ~/run_emulator.sh
```

将下面的文件内容，修改自己的路径之后，添加到到文件中

```shell
#!/bin/bash

# [修改这里] 你的 AOSP 源码绝对路径
AOSP_PATH="/home/chenhaochuan/aosp/android_10"

# [修改这里] 你编译时使用的 lunch target
LUNCH_TARGET="aosp_x86_64-eng"

echo "====== 正在准备启动 AOSP 模拟器 ======"

# 1. 进入源码目录
cd "$AOSP_PATH" || { echo "错误: 找不到源码目录 $AOSP_PATH"; exit 1; }

# 2 & 3. 导入环境并选择配置 (使用 bash 的 --rcfile 或直接在当前 shell 注入)
# 为了保证 emulator 启动后终端不关闭，并且环境变量生效，我们通过交互式 shell 启动
exec bash --init-file <(echo "source build/envsetup.sh && lunch $LUNCH_TARGET && emulator")
```

保存并且退出

```
1、保存，按下快捷键  Ctrl + O   字母欧
2、按下 回车键 Enter 切换到外部菜单
3、退出，按下快捷键  Ctrl + X   字母艾克斯
```

注意需要修改脚本为执行文件，使用下面的命令

```shell
chmod  +x  download.sh
```





### 第02节 配置环境变量

为了在任意位置，都可以快速的启动模拟器，我们可以配置环境变量，取别名，快速启动

使用下面的命令

```shell
nano  ~/.bashrc
```

在文件的末尾，添加下面的配置

```shell
# 添加别名启动模拟器 emulator

```

保存并且退出

```

2、按下 回车键 Enter 切换到外部菜单
3、退出，按下快捷键  Ctrl + X   字母艾克斯
```

刷新环境变量

```shell
source ~/.bashrc
```









### 第03节 虚拟化操作

当前我使用的是  `VMware Fusion`  虚拟机，使用的操作系统是  `MacOS Intel` 的操作系统。

如何开启虚拟化的操作？

在 Mac 的 访达（Finder） 当中，找到 Unbutu 的虚拟机文件

```
文档 ---> 虚拟机/Virtual Machines 的文件夹 ----> 找到后缀名称是  .vmwarevm

1、点击鼠标右键, 选择该虚拟机文件,  “显示包内容”  （Show Package Contents）
2、在弹出的文件夹当中, 找到一个后缀为  .vmx 的文件  （例如: Ubuntun.vmx）
3、右键点击这个 .vmx文件, 选择以文本的方式打开
4、在文件的末尾, 添加下面几行核心代码


vhv.enable = "TRUE"
nestedHVEnabled = "TRUE"
vpmc.enable = "TRUE"

添加完毕之后，保存文件，重新打开虚拟机。
```

如果在开启虚拟机的过程当中，出现了报错无法打开的情况下，你需要做如下的调整：

还是需要找到这个 `.vmx` 的文件， 将之前的内容，改为下面的两行内容，也就是说去掉  `vpmc.enable` 的选项

```
vhv.enable = "TRUE"
nestedHVEnabled = "TRUE"
```

添加完毕之后，保存文件，重新打开虚拟机。
