## 第一章 基础概述

### 第01节 介绍说明

对于小白来说，SystemUI 是一个庞然大物，但是我们需要掌握基础的操作

操作步骤如下：

```
1、定位源码
2、单独编译
3、动态推流
4、重启生效
```

学习起来就会非常快。 下面将会以  Android 10 （API 29）的 SystemUI 快速入门与实战指南







### 第02节 核心源码

在 Android 10 当中，SystemUI 的核心源码存放在下面的路径当中：

```
frameworks/base/packages/SystemUI
```

三个核心的模块

```
1、第一部分:
	A. 中文名称:  状态栏与下拉通知栏
	B. 英文名称:  StatusBar & Notification
	C. 所在路径:  src/com/android/systemui/statusbar/
	D. 具体作用:  负责管理手机最上方的状态栏（电量、时间、WiFi图标）以及下拉之后的通知列表
	
2、第二部分:	
	A. 中文名称:  快捷设置面板
	B. 英文名称:  Quick  Setting / QS
	C. 所在路径:  src/com/android/systemui/qs
	D. 具体作用:  下拉通知栏后，里面那些可以点击的“手电筒”、“蓝牙”、“飞行模式”等开关（称为 Title 瓷砖）
	
3、第三部分:
	A. 中文名称:  导航栏
	B. 英文名称:  Navigation  Bar
	C. 所在路径:  src/com/android/systemui/phone/NavigationBarView.java
	D. 具体作用:  屏幕底部的经典三键导航或全面手势条
```















## 第二章 操作流程

### 第01节 代码修改

在上述的路径中，找到指定位置，修改完毕代码即可。





### 第02节 编译环境

初始化编译环境，在源码的根目录下面执行下面的命令

源码根目录   `/home/chenhaochuan/aosp/android_10`

命令：

```shell
source  build/envsetup.sh

# 选择你的淡斑或者模拟器目标（例如:   lunch  aosp_x86_64-eng）
lunch
```





### 第03节 单独编译

单独编译 `SystemUI` 模块

使用 `mmm` 命令，单独编译指定目录下的模块

```shell
mmm  frameworks/base/packages/SystemUI
```

提示： 编译成功之后，终端会打印输出路径，Android 10 的 SystemUI 路径编译出来的结果通常是一个优化后的 APK

他一般位于  `out/target/product/<你的机型>/system/product/priv-app/SystemUI/SystemUI.apk `

我当前编译之后的路径

```
out/target/product/generic_x86_64/system/product/priv-app/SystemUI/SystemUI.apk 
```







### 第04节 推入设备

将编译之后的 APK，放入手机、模拟器

确保我的设备 手机或者模拟器已经连接 （`adb  devices` 可见） 并且是 `userdebug` 或者 `eng` 版本

执行下面的命令

```shell
# 获取 root 权限
adb  root

# 取消设备完整性校验，仅用于学习研究中使用，正式场合需要其他操作
adb  disable-verity

# 挂载系统分区为可读可写
adb  remount


# 将最新编译的 APK 推送到系统目录中, 覆盖原版
adb push out/target/product/generic_x86_64/system/product/priv-app/SystemUI/SystemUI.apk   /product/priv-app/SystemUI/SystemUI.apk


# 修改权限（防止 SELinux 因为安全策略拦截导致的 SystemUI 被拦截, 引发 SystemUI 无法加载、设备黑屏、卡死开机动画）
adb  shell  chmod  644  /product/priv-app/SystemUI/SystemUI.apk


# 清除缓存并且重启 SystemUI 进程
adb  shell  pkill  -f  com.android.systemui
```
