## 第一章 前言

提出问题

```
三个问题：
	1、为什么需要使用日志？
	2、为什么需要工具类？
	3、为什么需要自制工具类？
```

### 问题1：

```
问题：
	为什么需要使用日志?
	
回答：
	日志是开发测试过程中, 针对于业务逻辑使用, 新增的一种复现 bug 追踪 bug 的一种手段。
	通过日志的复现, 去检查当前的业务逻辑是否存在漏洞或者错误情况。
	
简述:
	日志是追踪复现问题的一种手段
```



### 问题2：

```
问题:
	为什么需要使用工具类?
	
回答:
	在我们采用日志的过程中, 会大量重复的使用日志, 对于重复的多次工作, 建议采用工具类, 快速完成日志的获取
	
简述:
	快速使用日志, 避免重复的操作
```

### 问题3：

```
问题：
	为什么需要自制工具类?
	
回答:
	市面上存在多种日志操作的工具类, 但是每种工具类, 可能存在隐形的 bug 或其他问题。这种情况下, 需要自己去定义工具类

简述:
	自己定义减少bug 可以自由DIY 实现日志处理
```







## 第二章 用法

### 1、定义工具类

```java

import android.util.Log;

import java.util.Formatter;

/****
 *
 * @apiNote 日志工具类
 */
public class CommonLogUtil {

    public static final int LEVEL_OPEN = 1;     // 开启
    public static final int LEVEL_VERBOSE = 2;  // 冗长
    public static final int LEVEL_DEBUG = 3;    // 调试
    public static final int LEVEL_INFO = 4;     // 信息
    public static final int LEVEL_WARN = 5;     // 警告
    public static final int LEVEL_ERROR = 6;    // 错误
    public static final int LEVEL_CLOSE = 7;    // 关闭

    // 初始化默认的 TAG
    private static final String TAG = "cosmo";
    // 初始化默认的等级
    private static final int currentLevel = LEVEL_OPEN;
    // 开启头信息
    private static final boolean isOpenHead = true;


    public static void verbose(String TAG, String message) {
        if (currentLevel > LEVEL_VERBOSE) {
            return;
        }
        if (isOpenHead){
            message =  formatLogHead() + message;
        }
        Log.v(TAG, message);
    }

    public static void verbose(String message) {
        verbose(TAG, message);
    }

    public static void debug(String TAG, String message) {
        if (currentLevel > LEVEL_DEBUG) {
            return;
        }
        if (isOpenHead){
            message =  formatLogHead() + message;
        }
        Log.d(TAG, message);
    }

    public static void debug(String message) {
        debug(TAG, message);
    }

    public static void info(String TAG, String message) {
        if (currentLevel > LEVEL_INFO) {
            return;
        }
        if (isOpenHead){
            message =  formatLogHead() + message;
        }
        Log.i(TAG, message);
    }

    public static void info(String message) {
        info(TAG, message);
    }

    public static void warn(String TAG, String message) {
        if (currentLevel > LEVEL_WARN) {
            return;
        }
        if (isOpenHead){
            message =  formatLogHead() + message;
        }
        Log.w(TAG, message);
    }

    public static void warn(String message) {
        warn(TAG, message);
    }

    public static void error(String TAG, String message) {
        if (currentLevel > LEVEL_ERROR) {
            return;
        }
        if (isOpenHead){
            message =  formatLogHead() + message;
        }
        Log.e(TAG, message);
    }

    public static void error(String message) {
        error(TAG, message);
    }

    private static String formatLogHead() {
        // 格式头信息
        String pattern = "[%s][%s.java:%d][%s]: ";
        // 获取线程名称
        String threadName = Thread.currentThread().getName();
        //--------------------------------------------------------------------
        StackTraceElement targetElement = new Throwable().getStackTrace()[3];
        // 获取类名称
        String[] classNameInfo = targetElement.getClassName().split("\\.");
        String className = classNameInfo[classNameInfo.length - 1];
        className = className.contains("$") ? className.split("\\$")[0] : className;
        // 获取当前代码行数
        int lineNumber = targetElement.getLineNumber();
        // 获取方法名称
        String methodName = targetElement.getMethodName();
        String[] methodNameInfo = methodName.split("\\$");
        methodName = methodName.contains("$") ? methodNameInfo[methodNameInfo.length - 1].substring(1) : methodName;
        // 格式化
        Formatter format = new Formatter().format(pattern, threadName, className, lineNumber, methodName);
        return format.toString();
    }
}
```

### 注意事项

```
日志的一些问题：性能的影响
虽然日志使用过程中, 能够快速帮助我们定位到问题所在地, 但是也会出现性能降低的问题,会进行大量的 IO 读写操作, 针对于 CPU 性能存在一定量的影响。
那么在使用的过程中, 需要有几点小的建议：
		1、日志需要精准, 能够通过当前的日志解决一类问题
		2、合理控制日志级别, 输出各类日志时, 考虑清楚日志是否需要
		3、正式环境中, 建议屏蔽掉日志信息, 可能存在日志包含有敏感信息的情况
```
