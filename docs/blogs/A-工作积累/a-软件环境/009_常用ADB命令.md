## 第一章 基础命令



### 第01节 常用命令

服务和设备

```shell
# 1. 查看已经连接的设备信息
adb  devices

# 2. 启动 adb 服务
adb  start-server 
 
# 3. 关闭 adb 服务
adb  kill-server
```

日志相关

```shell
# 调节日志缓存的大小
adb  logcat   -G  50M

# 保存日志到电脑文件中
adb  logcat  -v   time  >  C:\Users\cosmo\Destop\log.txt
```

安装应用

```shell
# 基础安装的命令
adb  install  C:\Users\cosmo\Destop\Demo.apk

# 强制安装的命令
adb  install  -r  C:\Users\cosmo\Destop\Demo.apk

# 其他配置参数以及含义
-l    	锁定该应用程序
-r		替换已经存在的应用程序, 也就是说 强制安装
-t		允许测试包
-s		把应用安装到 SDcard 上
-d		允许进行将见状, 也就是安装的手机上的带的版本低
-g		为应用程序授予所有运行时的权限
```

卸载应用

```shell
# 卸载应用程序
adb  uninstall  (apk的包名称)

# 卸载应用程序, 但是保留数据和缓存文件
adb  uninstall  -k   (apk的包名称)
```



















### 第02节 文件操作

文件拷贝

```shell
# 将文件从 Android设备 拷贝到 电脑
adb  pull   Android设备的路径     电脑的路径
adb  pull  /data/system/dropbox   C:\crash\dropbox

# 将文件从 电脑 拷贝到 Android设备
adb  push   电脑路径    Android设备路径
adb  push   C:\crash\dropbox\Demo.apk     /system/app/demo/
```

图片拷贝到 Android设备中，并且通知相册更新

```shell
# 第一步: 利用 push 操作将文件添加到相册中
adb  push  ./1.png   /sdcard

# 第二步：发送广播通知系统将图片添加到相册中
adb  shell  am  broadcast  -a  android.intent.action.MEDIA_SCANNER_SCAN_FILE  -d  file:///sdcard/1.png
```

增删文件的操作

```shell
# 删除 Android 设备上面的文件
adb  shell  rm   /system/app/demo/demo.apk

# 删除目录下面所有的 txt 文件
adb  shell  rm   /sdcard/hello/\*.txt

# 删除目录下面所有的 文件
adb  shell  rm  -rf  /sdcard/hello/\*.\*

# 删除 Android 设备上的文件夹
adb  shell  rm  -rf   /sdcard/hello

# 创建文件夹
adb  shell  mkdir  -p    /sdcard/hello
```

显示文件或搜索文件操作

```shell
# 显示该目录下面所有的文件
adb  shell  ls  /sdcard/

# 显示该目录下面所有的文件（包括权限、时间 等详细信息）
adb  shell  ls  -l    /sdcard/
adb  shell  ls  -al   /sdcard/

# 查找 搜索文件
adb  shell  文件夹名称  find  .  -name   文件名称
adb  shell  busybox     find  .  -name   "aaa.jpg"
```









### 第03节 设备信息

硬件信息

```shell
# 查看磁盘空间状况
adb  shell  df  -h

# 查看机器分辨率
adb  shell  wm  size

# 获取像素密度
adb  shell  wm  density
```

软件信息

```shell
# 获取 Android 系统版本
adb  shell   getprop   ro.build.version.release

# 查看 Android设备 是 32位 还是 64位
adb  shell   getprop   ro.product.cpu.abi
(说明: 64位系统 显示 arm64-v8a)

# 查看 Android设备 机器厂商
adb  shell  getprop   ro.product.brand

# 查看 Android设备 机器名称
adb  shell  getprop   ro.product.model

# 查看 Android设备 基础信息（SDK版本号,CPU类型,desity,机器名称,厂商等）
adb  shell   cat/system/build.prop
```











### 第04节 界面和APP

界面

```shell
# 查看当前的 Activity
adb   shell   dumpsys   activity   activities |  find   "Run"
```

获取APP信息

```shell
# 列出 Android设备 中安装的 所有 App 的包名称
adb   shell   pm  list  packages

# 列出系统应用的 所有系统包名称
adb   shell   pm  list  packages  -s

# 列出系统应用的 第三方应用包名称
adb   shell   pm  list  packages  -3

# 清除 app 数据 （根据包名称, 包名称必须存在正确才会提示 success）
adb   shell   pm  clear  com.cosmo.xx.xx
```













## 第二章 组件进程

### 第01节 界面相关

启动 Activity 不带 Action 的方式

```shell
# 快速启动
adb   shell   am   start   -n   com.xxx.xxx/com.xxx.xxx.MainActivity
```

启动 Activity 带有 Action 的方式

```shell
# 启动
adb   shell   am   start   -a   com.action.xxx.xxxx

# 参数说明
-a  ACTION
-d  DATA_URI
-t  MIME_TYPE
-c  CATEGORY
-e | --es     EXTRA_KEY   EXTRA_STRING_VALUE
--ez   		 ENTRA_KEY  EXTRA_BOOLEAN_VALUE
-e | --ei     EXTRA_KEY  EXTRA_INT_VALUE
-n  COMPONENT
-F  FLAGS
URI
```







### 第02节 服务相关

启动 Service

```shell
# 启动 Servivice 包名称和类名称
adb  shell  am  startservice  -n   com.xxx.xxx/com.xxx.xxx.MyService

# 启动 Service 带有 ACTION 的方式
adb  shell  am  startservice  -a   com.action.action.xxx
```

停止 Service

```shell
# 停止 正在运行的 Service
adb  shell  am  stopservice   com.xxx.xxx/com.xxx.xxx.MyService
```

查看 Service

```shell
# 方式一: 查看 Service 正在运行  (这里的 MyService 是服务的类名称, 建议使用类名称查找更加准确, 有内容返回说明正在运行)
adb  shell  dumpsys  activity  services   |  adb  shell  grep  MyService

# 方式二： 查看 Service 正在运行 (也可以判断 service 是否运行, 测试没有运行会返回 nothing 建议采用上面的方法)
adb  shell  dumpsys   activity  services  MyService

# 应用程序进程是否正在运行 （后台或者前台）
adb  shell  pidof   <package.name>
```







### 第03节  进程相关

查看进程

```shell
# 查看进程 （有时候智能查询少数几个, 原因不明）
adb  shell  ps

# 根据关键字查找进程 （进程名一般为包名称 com.xxx.xxx 输入 xxx 即可以查询, 包含 xxx 即可）
adb  shell  "ps  | grep  xxx"  
```

关闭进程

```shell
# 关闭进程 （根据进程 id,  id 通过上面的 ps 命令查找, 用此方法关闭, 进程 app 中 service 会立即重新启动）
adb  shell  kill  -9   进程ID

# 关闭进程 （根据包名称, 采用此方法关闭, 进程 app 中 service 不会立即启动）
adb  shell  am  force-stop   包名称
```

**案例2** 执行 adb shell 需要返回结果

```java
/**
 * Execute the command.
 *
 * @param commands        The commands.
 * @param isRooted        True to use root, false otherwise.
 * @param isNeedResultMsg True to return the message of result, false otherwise.
 * @return the single {@link CommandResult} instance
 */
public static CommandResult execCmd(final String[] commands,
                                    final boolean isRooted,
                                    final boolean isNeedResultMsg) {
    int result = -1;
    if (commands == null || commands.length == 0) {
        return new CommandResult(result, "", "");
    }
    Process process = null;
    BufferedReader successResult = null;
    BufferedReader errorResult = null;
    StringBuilder successMsg = null;
    StringBuilder errorMsg = null;
    DataOutputStream os = null;
    try {
        process = Runtime.getRuntime().exec(isRooted ? "su" : "sh");
        os = new DataOutputStream(process.getOutputStream());
        for (String command : commands) {
            if (command == null) continue;
            os.write(command.getBytes());
            os.writeBytes(LINE_SEP);
            os.flush();
        }
        os.writeBytes("exit" + LINE_SEP);
        os.flush();
        result = process.waitFor();
        if (isNeedResultMsg) {
            successMsg = new StringBuilder();
            errorMsg = new StringBuilder();
            successResult = new BufferedReader(
                    new InputStreamReader(process.getInputStream(), "UTF-8")
            );
            errorResult = new BufferedReader(
                    new InputStreamReader(process.getErrorStream(), "UTF-8")
            );
            String line;
            if ((line = successResult.readLine()) != null) {
                successMsg.append(line);
                while ((line = successResult.readLine()) != null) {
                    successMsg.append(LINE_SEP).append(line);
                }
            }
            if ((line = errorResult.readLine()) != null) {
                errorMsg.append(line);
                while ((line = errorResult.readLine()) != null) {
                    errorMsg.append(LINE_SEP).append(line);
                }
            }
        }
    } catch (Exception e) {
        e.printStackTrace();
    } finally {
        try {
            if (os != null) {
                os.close();
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        try {
            if (successResult != null) {
                successResult.close();
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        try {
            if (errorResult != null) {
                errorResult.close();
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
        if (process != null) {
            process.destroy();
        }
    }
    return new CommandResult(
            result,
            successMsg == null ? "" : successMsg.toString(),
            errorMsg == null ? "" : errorMsg.toString()
    );
}

/**
 * The result of command.
 */
public static class CommandResult {
    public int    result;
    public String successMsg;
    public String errorMsg;

    public CommandResult(final int result, final String successMsg, final String errorMsg) {
        this.result = result;
        this.successMsg = successMsg;
        this.errorMsg = errorMsg;
    }

    @Override
    public String toString() {
        return "result: " + result + "\n" +
                "successMsg: " + successMsg + "\n" +
                "errorMsg: " + errorMsg;
    }
}
```
