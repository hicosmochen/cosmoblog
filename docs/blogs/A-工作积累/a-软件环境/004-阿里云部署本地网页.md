## 第一章 简述

### 第01节 基础说明

如何将本地的静态网页（Html 、CSS、JS ）发布到网络上面，通过浏览器进行访问。

基础原理，借助于 Nginx 服务器, 发布 网页信息。

基础的操作步骤如下：

```
1、需要准备本地的 网页资源文件
2、在服务端的创建 部署的文件夹
3、将本地的网页资源文件，上传至 服务端的部署文件夹下面
4、通过 Nginx 服务器，配置服务端的路径，进行访问
5、需要重新加载 Nginx 服务
```



### 第02节 快捷指令

上传本地文件到远程服务端

```
指令
	scp  -r   本地文件夹的路径  root@121.199.34.5:远端文件夹的地址

例如:
	scp  -r  /Users/chenhaochuan/Desktop/lunbo  root@121.199.34.5:/root/cosmodemo
```

修改文件夹及子文件权限

```
chmod  -R  777  文件夹的名称
```

修改配置 Nginx 信息

```
nginx  -t

vim  /etc/nginx/nginx.conf
```

重新发布

```
nginx  -s   reload
```

## 第三章 Nginx配置文件

```
# NGINX 配置文件

# 定义用户和工作组
user root;  # 根据实际用户修改
worker_processes auto;

# 日志设置
error_log /var/log/nginx/error.log;
pid /var/run/nginx.pid;

events {
worker_connections 1024;  # 定义最大连接数
}

http {
include /etc/nginx/mime.types;  # 引入 MIME 类型
default_type application/octet-stream;

# 日志格式
access_log /var/log/nginx/access.log;

# 发送文件相关设置
sendfile on;
tcp_nopush on;
tcp_nodelay on;
keepalive_timeout 65;
types_hash_max_size 2048;

# Gzip 设置
gzip on;
gzip_types text/plain application/xml text/css application/javascript;
gzip_min_length 256;

server {
listen 80;  # 监听 80 端口
server_name hicosmo.vip;  # 替换为您的域名或 IP 地址

# root /root/application/webapp;  # 设置根目录

location /test {
alias /root/cosmolunbo/lunbo;  # 使用 alias 指令
index index.html;  # 设置默认首页文件
try_files $uri $uri/ =404;  # 尝试访问文件，如果不存在则返回 404
}

# 可选：设置默认的错误页面
error_page 404 /404.html;  # 自定义 404 页面
location = /404.html {
internal;  # 仅限内部访问
}

# 可选：启用日志
access_log /var/log/nginx/testdemo_access.log;
error_log /var/log/nginx/testdemo_error.log;
}
}
```











## 第四章 遇到的问题

### 第01节  交换文件的问题

出现交换文件  swap 文件出现的情况下。删除掉 指定目录下的 swap 文件

```
rm  -rf   文件名称
```

> -rf  的说明:  r 表示递归， f 表示暴力删除







### 第02节 Pid失效的问题

报错信息如下：

```
nginx: [error] invalid PID number "" in "/var/run/nginx.pid"

Job for nginx.service failed because the control process exited with error code.
```

修复方案：重启两个端口

```
sudo  fuser  -k   80/tcp

sudo  fuser  -k   443/tcp
```







### 第03节 403的问题

如果访问出现 403 的情况，那么可以从以下的几个方面排查

**方向01**

```
问题: 
	文件夹的权限是否给予?
	
解决:
	cd ~
	chmod  -R  777  application
```





**方向02**

```
问题:
	文件夹路径是否错误?
	
解决:
	注意这个路径是 index.html 的路径, 拷贝的时候，容易路径写错。
	alias /root/cosmolunbo/lunbo;
```



**方向03**

```
问题:
	之前 Nginx 的配置文件错误?
	
解决:
	复制上面的配置文件信息, 到配置文件中。
```
