## 第一章 生成so文件

### 第01节 生成头文件

基础描述

```
1. 创建一个普通的 Android 项目工程或者 Java项目工程即可。
2. 创建一个 Java 文件, 该Java文件类当中包含有 native 格式的方法。
3. 使用指令生成头文件。
```

核心操作指令

```
// 需要在 main/Java 目录下面使用下面的指令, 就可以得到头文件信息了。

javac  -h  .   包名称1/包名称2/包名称3/类名称.java
```

### 第02节 配置JNI文件夹

基础描述

```
在安卓项目 src/main 下面创建 jni 文件夹。 文件夹里面主要包含如下四个文件:
1、Android.mk
2、Application.mk
3、需要操作的C或者CPP文件
4、生成的头文件.h
```

四个文件的代码如下：

**Android.mk 文件**

```makefile
# 设置当前模块的路径
LOCAL_PATH := $(call my-dir)

# 清除之前定义的变量（除了 LOCAL_PATH）
include $(CLEAR_VARS)

# 定义模块的名称 这里的模块名是 calc，最终生成的动态库文件名为 libcalc.so（Android 会自动添加 lib 前缀和 so 后缀）
LOCAL_MODULE := calc

# 指定模块的源文件, 如果有多个源文件，可以用空格分隔, 例如:  LOCAL_SRC_FILES := calc.cpp utils.cpp
LOCAL_SRC_FILES := calc.cpp

# 指定模块的类型为动态链接库（.so 文件）
include $(BUILD_SHARED_LIBRARY)
```



**Application.mk 文件**

```makefile
# 指定 NDK 编译时目标的最低 Android API 级别
APP_PLATFORM := android-24

# 指定 NDK 编译的目标 CPU 架构
APP_ABI := armeabi-v7a arm64-v8a
```



**自己编写的cpp文件**

```c++
#include "com_baby_CalcBabyAge.h"
JNIEXPORT jint Java_com_baby_CalcBabyAge_compareAge
        (JNIEnv *env, jobject jclass, jint ageX, jint ageY){
    return ageX+ageY;
}
```

**生成的头文件**  

头文件是自己生成的不需要我们进行编译或修改的操作，我们需要操作的是 CPP或C文件

```c++
/* DO NOT EDIT THIS FILE - it is machine generated */
#include <jni.h>
/* Header for class com_baby_CalcBabyAge */

#ifndef _Included_com_baby_CalcBabyAge
#define _Included_com_baby_CalcBabyAge
#ifdef __cplusplus
extern "C" {
#endif
/*
 * Class:     com_baby_CalcBabyAge
 * Method:    compareAge
 * Signature: (II)I
 */
JNIEXPORT jint JNICALL Java_com_baby_CalcBabyAge_compareAge
  (JNIEnv *, jobject, jint, jint);

#ifdef __cplusplus
}
#endif
#endif
```

### 第03节 编译生成SO文件

基础描述

```
1、需要确认一下，当前的ndk路径
2、需要确认一下，当前编译的最低版本
3、在 Java 目录下面, 执行编译指令 ndk-build 得到so文件
```

**确认当前 NDK 的路径**

```
主要在 Android 项目的 local.properties 文件夹当中, 需要定义下面的内容:
一般情况下, NDK 在我们的 SDK 工具更新下载之后, 应该存在于 sdk 的下一级子目录文件夹ndk当中

ndk.dir=当前NDK所在的路径
```

## 第二章 使用so文件

### 第01节 导入准备

操作步骤

```
1、创建文件夹, 在 main 文件夹下面创建 jniLibs 文件夹，导入相关架构的 so 库
2、build.gradle 当中, 引入配置相关
```



**创建文件夹并且导入so库**

基础描述

```
1、创建文件夹
	文件夹名称:   	jniLibs
	文件夹所在路径:  项目目录下面的 src/main 文件夹下面	
	
2、路径展示格式如下:
	---- src
	----------- main
	------------------- java
	------------------- jniLibs
	------------------- res
	
3、导入相关的 so库文件
	将前面创建的  arm64-v8a、 armeabi-v7a  相关的文件导入
```

**在 build.gradle 当中引入相关的配置**

基础描述

```groovy
// 在 App 下面的 build.gradle 当中, 主要引入的配置存在两点内容：
	
1、 ndk { ... }
2、 sourceSets { ... }
```

核心代码如下:

```groovy
android {

    defaultConfig {

		// 上面的内容省略...

        ndk {
            abiFilters 'armeabi-v7a', 'arm64-v8a', 'x86', 'x86_64' // 根据你提供的.so文件选择
        }
    }

    sourceSets {
        main {
            jniLibs.srcDirs = ['src/main/jniLibs']
        }
    }
    
}
```









### 第02节 包类同名

操作步骤

```
1、建包和建类。 
	1.1 注意这里的包名称和类名称, 强烈建议与 之前生成JNI SO库的名称保持一致
	1.2 例如: 之前建立JNI的时候, 当前的 SO库指向的地址是 com.baby.CalcBabyAge 那么我们也创建相同的包和类
	
2、类中引入 System.loadLibrary("")  以及 相关的 native 格式方法
	2.1 需要加入静态代码库, 引入 System.loadLibrary("...");    
	2.2 定义和之前相同的 native 格式的方法。
	
备注:
	System.loadLibrary("")  参数需要 "掐头去尾"
	例如: 我们的保存在 jniLibs/arm64-v8a/libcalc.so  
	这里 so 库的名称是 libcalc.so
	那么我们在使用的过程中, 需要去掉前面的 lib 也要去掉后面的 .so 只保留中间的 calc 
	因此, 这里的写法就是 System.loadLibrary("calc")
```

案例代码

```java
package com.baby;

public class CalcBabyAge {

    static {
        System.loadLibrary("calc");
    }

    public native int compareAge(int ageA, int ageB);
}

```







### 第03节 直接使用

在Activity当中编写代码如下：

```java
import androidx.appcompat.app.AppCompatActivity;

import android.os.Bundle;
import android.util.Log;
import android.widget.TextView;

import com.baby.CalcBabyAge;

public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        TextView textView = findViewById(R.id.tv);

        int result = 0;

        //==========================================================================================================
        // 下面的用法,是作为一个非常简单的使用案例,来实现此功能
        // 在正常使用过程中, 建议中间封装一层接口, 便于处理这层异常问题
        try {
            CalcBabyAge baby = new CalcBabyAge();
            result = baby.compareAge(10, 20);
        } catch (UnsatisfiedLinkError e) {
            Log.e("MainActivity", e.toString());
        }
        //==========================================================================================================

        textView.setText(String.valueOf(result));
    }
}
```

备注说明

```java
如果在使用 JNI 的过程中, 出现了上面的异常情况。

异常类型 UnsatisfiedLinkError

需要分析以下几点内容:
	1、JNI 接口, 对应的 "包名称" 是否正确?
	2、JNI 接口, 对应的 "类名称" 是否正确?
	3、JNI 接口, 对应的 "SO库"  是否正常进行了 掐头去尾操作?
```
